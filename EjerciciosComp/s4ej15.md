# Ejercicio 15: Mejores Practicas

## 1) Por que es importante la preparacion de datos
La preparacion de datos mejora la calidad de entrada del modelo. Si los datos tienen errores, valores faltantes, formatos inconsistentes o escalas muy distintas, el modelo aprende patrones incorrectos y su desempeno cae.

Beneficios clave:
- Reduce ruido y sesgos evitables.
- Mejora la estabilidad y precision del entrenamiento.
- Hace comparables variables en diferentes escalas.
- Permite pipelines reproducibles.

## 2) Que es data leakage y como evitarlo
Data leakage ocurre cuando el modelo recibe informacion que en produccion no estaria disponible al momento de predecir. Esto infla metricas en validacion y da una falsa sensacion de buen desempeno.

Ejemplos comunes:
- Escalar o imputar usando todo el dataset antes de train/test split.
- Usar variables que contienen informacion futura o del target.
- Ajustar hiperparametros usando el conjunto de prueba.

Como evitarlo:
- Separar train/test al inicio del flujo.
- Ajustar transformaciones solo con train y aplicar a test.
- Usar Pipeline de sklearn para encapsular preprocesamiento + modelo.
- Reservar el test para evaluacion final.

## 3) Diferencia entre datos de entrenamiento y prueba
- Entrenamiento: se usa para ajustar parametros del modelo.
- Prueba: se usa solo para evaluar generalizacion en datos no vistos.

Idea principal:
Si un modelo rinde bien en entrenamiento pero mal en prueba, hay sobreajuste. El conjunto de prueba estima el rendimiento esperado en produccion.
